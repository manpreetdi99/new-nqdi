Advanced Rebase Workflows

Understanding Rebase vs Merge

Rebase replays commits from one branch onto another, creating a linear history. Unlike merge, which creates a merge commit, rebase rewrites your branch's history.

Golden Rule: Never rebase commits that have been pushed to a shared repository, unless you coordinate with your team. Rebase rewrites history, which will cause conflicts for anyone who has pulled those commits.

Basic Rebase: Linear History

$ git rebase main


# Replays current branch's commits on top of main
# Before:
#   main:     A---B---C
#   feature:           D---E

# After:
#   main:     A---B---C
#   feature:           A---B---C---D'---E'

Interactive Rebase (rebase -i)

The most powerful Git command for rewriting history. Allows you to reorder, squash, edit, or delete commits.

Scenario: Clean up last 5 commits
$ git rebase -i HEAD~5


# Opens editor with:
pick a1b2c3d Fix typo
pick b2c3d4e Add validation
pick c3d4e5f Refactor parser
pick d4e5f6g WIP: debugging
pick e5f6g7h Fix edge case

# Change to:
pick a1b2c3d Fix typo
squash b2c3d4e Add validation
squash c3d4e5f Refactor parser
reword d4e5f6g WIP: debugging
fixup e5f6g7h Fix edge case

Interactive Rebase Commands

Command Effect Use Case
pick Keep commit as-is Default, commits you want to keep
reword Keep commit, edit message Fix typos in commit messages
edit Pause, let you amend commit Split commits, add files, change content
squash Meld into previous commit Combine related commits, preserve messages
fixup Squash, discard message Combine WIP or debug commits
break Pause rebase at this point Manual testing between commits
drop Delete commit entirely Remove WIP or bad commits
exec Run shell command Verify each commit passes tests

Advanced: Rebase with --onto

Rebase a branch onto a different base than its merge-base. Extremely useful for moving branches or extracting commits.

Move commits to different base
$ git rebase --onto main feature~2 feature


# Takes the last 2 commits of feature
# (from feature~2 to feature)
# and replays them on top of main

# Before:
#   main:     A---B---C
#   feature:        D---E---F

# After:
#   main:     A---B---C
#   feature:           A---B---C---E'---F'
#   (D' is left behind)

Rebase with Autostash

$ git rebase --autostash main


# Automatically stashes uncommitted changes
# Applies them after rebase completes
# No need to manually stash beforehand

Handling Conflicts During Rebase

$ git rebase main
CONFLICT (content): Merge conflict in app.js
# Edit files to resolve...
$ git add app.js
$ git rebase --continue

# OR abort entirely
$ git rebase --abort
Be Careful: git rebase --continue will continue with the next commit. If there are still conflicts, you'll be prompted again. Use git status to see what's unresolved.
